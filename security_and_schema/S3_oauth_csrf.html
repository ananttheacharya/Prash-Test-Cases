[S3] OAuth CSRF Vulnerability Test

The backend /auth/github/login endpoint redirects to GitHub but does not include a state parameter.
This allows an attacker to initiate the OAuth flow themselves, stop before the redirect back to Drufiy, and trick a victim into completing the flow. The victim's browser will be authenticated as the attacker.

Click here to claim your prize! (Victim clicks this)

Because the backend doesn't check a state cookie/session against the callback, it blindly accepts ATTACKERS_OAUTH_CODE and logs the victim in as the attacker.
